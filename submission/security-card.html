Framework-enforced human gate

Every mutating action pauses on ADK require_confirmation. The gate is code, not a prompt the model can talk itself out of.

Allow-lists that fail closed

Server-side bounds on every remediation: replica band, known-good versions, managed flags. An out-of-bounds action is refused even if approved.

Telemetry is never an instruction

All Dynatrace data is treated as untrusted evidence to summarize. A poisoned log line gets flagged as an anomaly, not obeyed.

Abuse-bounded public demo

Per-IP rate limits, a single-active-run guard, a 300s approval timeout, and one pinned instance cap worst-case model spend.

Every decision on the record

An append-only ledger audits approvals and refusals alike, written back to Dynatrace with an honest sent vs verified badge.

Locked transport + zero secrets

CSP, HSTS, and a CORS lock to one origin; secrets only in Secret Manager; the repo and all 85 commits scan clean.

Graded, not vibes: the same audit philosophy as the eval suite. The one open surface (the demo target's admin API) is an accepted, documented trade-off with the hardening flag already shipped, and SECURITY.md states it plainly.
